DROID SURVAIR PORTAL - MOVE / NEW PC SETUP

This folder is the clean code package.
It excludes generated heavy data:
- backend venv
- frontend node_modules
- frontend dist/build
- converted Project_Data tiles / 3D models / point clouds
- git history

NEW PC REQUIRED INSTALLS
1. Python 3.10 / 3.11 / 3.12
   During install, enable "Add Python to PATH".

2. Node.js LTS
   This includes npm.

3. QGIS 3.44.8 or compatible OSGeo4W install
   Default expected path:
   C:\Program Files\QGIS 3.44.8\OSGeo4W.bat
   Used for gdal_contour and GDAL fallback tools.

4. Optional for Potree point cloud conversion:
   C:\PotreeConverter\PotreeConverter.exe
   If not installed, normal portal and TIFF/3D Tiles ZIP upload can still work,
   but LAS/LAZ to Potree web conversion will not.

SETUP STEPS
1. Copy this full folder to the new PC.
2. Open droid_manager.bat.
3. Choose option 1: One Click Install / Repair.
4. After success, choose option 3: Start the Portal.
5. Open:
   http://localhost:5173

DATA NOTE
Do not copy old converted Project_Data if you want a clean move.
On the new PC, upload/import fresh:
- Cesium 3D Tiles ZIP models
- TIFF Ortho / DTM / DSM
- LAS / LAZ point clouds
- KML / GeoJSON / DWG

The portal will recreate Project_Data output folders automatically.

OPTIONAL ENV SETTINGS
Backend file:
backend\.env

Useful settings:
LOCAL_DATA_PATH=<this folder>/Project_Data
TIFF_TILE_BUDGET_MB=100
TIFF_TILE_MAX_ZOOM_LIMIT=20
OSGEO4W_BAT=C:\Program Files\QGIS 3.44.8\OSGeo4W.bat
POTREE_CONVERTER_EXE=C:\PotreeConverter\PotreeConverter.exe
